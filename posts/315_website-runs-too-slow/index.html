网站的访问速度很慢

Oct 6, 2009

以下内容恢复自 Wordpress 时期的数据库备份，内容已经严重过期，仅留作纪念。

最近我在优化网站性能的过程中遇到了一个问题。

问题

我安装好Google XML Sitemaps就直接打开用了。刚开始一切看起来完全正常，直到网站打开变得巨慢。

排除

我的网站托管在美国的一家主机商，平时服务器的Ping大概在300ms左右。即使在高峰期，Ping值也没有出现明显波动。这说明服务器的网络并不是问题的根源。

排查

首先我逐一关闭所有插件，发现网站速度确实有所改善。这说明问题可能出在某个插件上。

最终我锁定了Google XML Sitemaps这个插件。关闭它后，网站速度恢复正常。

总结

我发现问题在于主机商对内存和CPU的限制。只要资源使用超过阈值，就会导致网站出现类似宕机的情况。打开Google XML Sitemaps限制内存占用的功能就好了。

我等下得找找看有没有好的缓存插件，看样子数据库的查询也不是很顺利，请大家一定要注意插件的使用。
