GhostXP 无法使用远程桌面的问题

Oct 28, 2009

以下内容恢复自 Wordpress 时期的数据库备份，内容已经严重过期，仅留作纪念。

最近在使用时，我发现很多GhostXP系统的远程桌面功能默认是被屏蔽掉的，甚至连注册表中启用远程桌面的权限都进行了严格的限制。

虽说这么做是为了安全，但个人还是挺喜欢这个功能的。至少在把另一台电脑当作家庭影音服务器的时候，能用远程连接进行管理会方便很多。

所以，我在这里为大家提供一个解决这个问题的方法。

开放注册表权限

首先，我们需要打开系统的注册表编辑器，在开始菜单的“运行”里输入“regedit”并点确定。

接着，在注册表路径中依次展开，找到 [HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Enum\Root\RDPDR] 这一项。

然后，在左侧的“RDPDR”目录上点击鼠标右键，选择“权限”，在弹出的安全窗口中，把“everyone”用户的“完全控制”选项勾选为允许。

导入注册表文件

权限修改好之后，需要导入一段关键的注册表文件。请将本文最底部的代码全部复制，保存成一个名为 remote.reg 的文本文件，然后双击运行将其导入到系统中。

启动相关系统服务

导入完成之后，再次打开开始菜单的运行，输入“services.msc”打开系统的服务管理窗口。

在长长的服务列表中，找到名为“Remote Desktop Help Session Manager”以及“Telnet”的这两个服务。

将它们的启动类型手动修改为“自动”运行。全部设置完毕后，只需重启电脑即可完成配置。

附：注册表代码

以下是需要另存为 remote.reg 文件的全部内容：

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Enum\Root\RDPDR\0000]
"ClassGUID"="{4D36E97D-E325-11CE-BFC1-08002BE10318}"
"Class"="System"
"HardwareID"=hex(7):52,00,4f,00,4f,00,54,00,5c,00,52,00,44,00,50,00,44,00,52,\
       00,00,00,00,00
"Driver"="{4D36E97D-E325-11CE-BFC1-08002BE10318}\\0030"
"Mfg"="(标准系统设备)"
"Service"="rdpdr"
"DeviceDesc"="终端服务器设备重定向器"
"ConfigFlags"=dword:00000000
"Capabilities"=dword:00000000
