DB Cache Reloaded 一个顶俩

Nov 1, 2009

以下内容恢复自 Wordpress 时期的数据库备份，内容已经严重过期，仅留作纪念。

以前我们常常会用到两款缓存插件：一个是专门用来缓存侧边栏的 WP Widget Cache，另一个则是提供页面缓存（但貌似不包含边栏）的 WP Super Cache。

为什么现在不需要它们了？

原因很简单，因为 DB Cache Reloaded 插件出现了！

这款插件有着更加高效的工作原理，它直接对数据库查询进行缓存。相比于以前的方案，它带来了几个让人无法拒绝的优点：

  • 它不需要占用更多的服务器空间来保存各种各样的 Cache 文件。
  • 它不会和 AJAX 类插件、AutoOptimise 插件，或者是 WPPostView 产生冲突。
  • 最省心的是，它完全不需要你去修改烦人的 .htaccess 文件。

如果你心动了，可以直接去 WordPress 官方插件库搜索“DB Cache Reloaded”进行下载替换。

附：如何彻底删除 WP Super Cache

如果你决定换用新插件，这里附带一下彻底删除 WP Super Cache 的方法。因为这款旧插件比较“顽固”，仅仅在后台停用是不够的：

  1. 首先，在 Super Cache 的设置页面里把现有的缓存全部清空，然后再到插件列表里把它禁用掉。
  2. 因为 Super Cache 修改了你的 .htaccess 文件，所以你需要手动修复它：打开你的 .htaccess 文件，把 BEGIN Super Cache 和 END Super Cache 之间的内容全部删除。
  3. 进入你的 wp-content 目录，删除里面的 advanced-cache.php 和 wp-cache-config.php 这两个文件。
  4. 顺手把 wp-content 目录下的 cache 整个文件夹也删掉。
  5. 最后，到 plugins 目录中，把 wp-super-cache 文件夹彻底删除。

按照以上步骤，你就可以把旧的缓存插件清理得干干净净啦！
