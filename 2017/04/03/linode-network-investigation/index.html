Los Angeles / San José 入口测试报告

Apr 3, 2017

最近打算把 Tokyo 区的 Linode 往美国迁移。

因为 Tokyo 区的网络状况已经有很长一段时间不稳定了，而且 Linode 一月二月间已经提示我三次因为硬件问题要重启母鸡了，每次提醒至少会开个工单跟我说一声，结果前两天我的 VPS 突然就被重启了，什么提示都没收到 …

简直不能忍。

然后我建了两个 Newark 的 Linode 想找个好点的 IP 段，结果刚好这两个 Linode 的 IP 段都不一样，路由也有一跳是不一样的，这一跳是美国的海底光缆入口，所以我很无聊地在这两台 Linode 上起了 Collectd 收集了 24 小时的数据来比较哪个好一点（笑）。

注意：此报告仅供消遣；时间有效性较短，不知道什么时候路由会变；测试用的节点也只经过了上海和青岛两处海底光缆出口。

Collectd 24 小时 Ping 测试

测试使用了三个节点，homesrv 位于魔都，pi_beta 在浙江某地，localhost 是青岛阿里云 VPS。

  • 两者丢包率几乎一致，说明丢包只跟出口有关
  • San José 入口的稳定性平均比洛杉矶好一点
